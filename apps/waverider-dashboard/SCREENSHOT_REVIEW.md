# WaveRider Dashboard Screenshot Review

Purpose: visual review gate for the dashboard prototype.

Required visual target:

- Four top hero cards.
- Dual-arc hero gauges.
- Large outer reality arc.
- Smaller inner actionability arc.
- Two numeric readouts per hero card.
- Light translucent glass dashboard shell.
- Wave-I color-wheel accent family.
- Visible MOCK / VISUAL PROTOTYPE boundary.

Pass conditions:

- Four hero cards are visible in desktop layout.
- Hero gauges show two arcs, not one generic gauge.
- Text is readable at screenshot scale.
- Mini metrics do not overpower the hero row.
- Lower panels support the dashboard without overwhelming it.
- Footer stays in MOCK / VISUAL / DORMANT posture.

Block conditions:

- A hero card loses the dual-arc gauge structure.
- The mock boundary is hidden or removed.
- The UI implies broker connection.
- The UI implies private account data.
- The UI implies trade execution.
- The page shifts back to a dark generic dashboard.
